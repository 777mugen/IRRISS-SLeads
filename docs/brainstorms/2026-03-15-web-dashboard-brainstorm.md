# Web Dashboard 设计文档

## What We're Building

为 IRRISS-SLeads 系统开发一个综合型 Web Dashboard，集成批处理监控、DOI 查询、数据分析、配置管理、CSV 导入导出等功能。采用 FastAPI + Jinja2 + HTMX 技术栈，实现快速开发和流畅的用户体验。

Dashboard 主要服务于内部用户（开发人员和销售团队），提供实时的批处理状态监控、线索查询、数据分析和销售反馈收集功能。

## Why This Approach

### 技术栈选择：FastAPI + Jinja2 + HTMX

**为什么选择这个技术栈**:
- ✅ **快速开发**: 前后端统一在 Python，无需学习 JavaScript 框架
- ✅ **轻量级**: 无需前后端分离，部署简单
- ✅ **流畅交互**: HTMX 实现局部刷新，体验接近 SPA
- ✅ **团队匹配**: 1-2 人小团队，专注后端逻辑
- ✅ **内部使用**: SEO 不重要，服务器端渲染足够

**为什么不用 Vue 3**:
- ⚠️ 前后端分离增加复杂度
- ⚠️ 需要额外的构建步骤和部署配置
- ⚠️ 学习曲线较陡（需要学习 JavaScript/TypeScript）
- ⚠️ 对于内部工具，过度设计

### 页面结构：单页多标签

**为什么选择单页多标签**:
- ✅ **导航简单**: 所有功能一目了然
- ✅ **无刷新切换**: HTMX 实现标签切换无刷新
- ✅ **开发快速**: 共享页面框架，减少重复代码
- ✅ **用户体验**: 不需要页面跳转，体验更流畅

**为什么不用多页面导航**:
- ⚠️ URL 管理复杂
- ⚠️ 页面跳转体验不够流畅
- ⚠️ 对于内部工具，单页更合适

## Key Decisions

### 决策 1: 不实现用户登录

**原因**: 内网访问，暂不需要权限控制

**未来考虑**: 如果需要对外展示，可以添加简单的 Basic Auth

### 决策 2: 销售反馈字段使用现有 Feedback 表

**字段映射**:
```python
class Feedback(Base):
    accuracy: str          # 线索准确性（好/中/差）
    demand_match: str      # 需求匹配度（好/中/差）
    contact_validity: str  # 联系方式有效性（好/中/差）
    deal_speed: str        # 成交速度（好/中/差）
    deal_price: str        # 成交价格（好/中/差）
    notes: str             # 销售备注
```

**原因**: 已在 `src/db/models.py` 中定义，无需修改数据库结构

### 决策 3: 关键词配置通过 YAML 文件管理

**配置文件**: `config/keywords.yaml`

**原因**: 
- ✅ 已有配置文件，无需重新设计
- ✅ YAML 格式易于阅读和编辑
- ✅ 通过 Web 界面编辑后，直接保存到 YAML 文件

**注意事项**: 编辑后需要重启服务才能生效（或实现热重载）

### 决策 4: 评分规则通过 YAML 文件展示

**配置文件**: `config/scoring_paper.yaml`

**展示方式**: 
- ✅ 可视化展示评分逻辑（表格或树状图）
- ✅ 显示每个字段的加分规则
- ⚠️ 暂不支持 Web 界面修改（需要手动编辑 YAML）

**未来考虑**: 可以添加 Web 界面修改功能

### 决策 5: CSV 导入导出使用 Pandas

**原因**:
- ✅ Pandas 已安装，处理 CSV 很方便
- ✅ 支持字段映射和数据清洗
- ✅ 可以处理大文件（流式读取）

**导出格式**:
```csv
DOI,对应作者,邮箱,地址,机构,评分,等级,创建时间
10.1136/jitc-2025-014040,Jumei Shi,shijumei@tongji.edu.cn,...
```

**导入格式**:
```csv
DOI,线索准确性,需求匹配度,联系方式有效性,成交速度,成交价格,备注
10.1136/jitc-2025-014040,好,中,好,差,中,客户对产品感兴趣
```

## Open Questions

### 问题 1: 批处理实时刷新频率？

**选项**:
- A) 每 30 秒自动刷新
- B) 每 1 分钟自动刷新
- C) 手动刷新按钮

**建议**: 每 30 秒自动刷新 + 手动刷新按钮

### 问题 2: CSV 上传后的处理流程？

**需要确认**:
- 上传后是否立即更新到数据库？
- 还是先预览，用户确认后再更新？
- 如果 DOI 不存在，如何处理？

**建议**: 
1. 上传后先预览解析结果
2. 显示匹配到的 DOI 数量
3. 用户确认后再更新到数据库

### 问题 3: 机构分布地图的数据来源？

**需要确认**:
- 使用 address_cn 字段解析城市？
- 需要正则表达式提取城市名称？
- 如何处理无法识别的地址？

**建议**: 
1. 从 address_cn 字段提取城市名称
2. 使用预定义的城市列表进行匹配
3. 无法识别的地址标记为"未知"

### 问题 4: 图表库选择？

**选项**:
- A) Chart.js（轻量级，适合基础图表）
- B) ECharts（功能丰富，支持中国地图）
- C) Plotly（交互性强，但较重）

**建议**: ECharts（支持中国地图 + 丰富的图表类型）
