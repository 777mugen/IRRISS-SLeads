## 🌐 系统概述

### 什么是 IRRISS-SLeads？

**IRRISS-SLeads** 是一个**销售线索发现系统**，专门用于：
- 从 PubMed 搜索学术论文
- 提取论文通讯作者信息（姓名、邮箱、电话、机构）
- 自动评分和分级（A/B/C/D/E）
- 管理和导出销售线索

### 技术架构

```
前端: FastAPI + Jinja2 + HTMX + Tailwind CSS + ECharts
后端: Python 3.14 + SQLAlchemy + AsyncPG
数据库: PostgreSQL
AI API: Zhipu GLM-5 (批量处理)
爬虫: Jina Reader API + PubMed Entrez API
```

### 核心组件

1. **Web Dashboard** - 可视化管理界面
2. **批处理系统** - 批量提取论文作者信息
3. **查询系统** - 按 DOI 查询论文
4. **分析系统** - 统计和数据分析
5. **导出系统** - CSV 导入/导出
6. **配置管理** - 关键词和评分规则

## 📋 操作流程

### 流程 1: 提取新论文

**目标**: 从 PubMed 提取 1000 篇论文的作者信息

**步骤**:

```bash
# Step 1: 启动 Web Dashboard（如果未运行）
nohup .venv/bin/uvicorn src.web.main:app --host 0.0.0.0 --port 8000 > logs/web.log 2>&1 &

# Step 2: 启动提取任务（后台运行）
nohup .venv/bin/python scripts/extract_papers_optimized.py --count 1000 > logs/extract_1000.log 2>&1 &

# Step 3: 监控进度（方式 1: 命令行）
tail -f logs/extract_1000.log

# Step 3: 监控进度（方式 2: 查看进度文件）
cat tmp/extraction_progress_*.json | jq .

# Step 4: 访问 Web Dashboard 监控
open http://localhost:8000/batch/monitor

# Step 5: 等待完成（40-80 分钟）
# 完成后会有批处理任务 ID

# Step 6: 监控 Zhipu 批处理状态
.venv/bin/python scripts/check_batch_status.py <batch_id>

# Step 7: 处理批处理结果
.venv/bin/python scripts/process_batch_results.py <batch_id>

# Step 8: 查看提取结果
open http://localhost:8000/query/search
# 输入 DOI 查询结果
```

**预计时间**: 40-80 分钟（1000 篇论文）

**注意事项**:
- ✅ 后台运行，不会超时
- ✅ 每 5 篇保存进度
- ✅ 可随时中断并恢复
- ⚠️ 需要 PostgreSQL 数据库运行

## 🔍 故障排查

### 问题 1: Web Dashboard 无法访问

**症状**: 浏览器无法打开 http://localhost:8000

**排查步骤**:

```bash
# Step 1: 检查进程是否运行
ps aux | grep uvicorn

# Step 2: 检查端口是否被占用
lsof -i :8000

# Step 3: 检查日志
cat logs/web.log | tail -50

# Step 4: 手动启动查看错误
.venv/bin/uvicorn src.web.main:app --host 0.0.0.0 --port 8000
```

**常见原因**:
- ✅ 服务未启动 → 启动服务
- ✅ 端口被占用 → kill 占用进程
- ✅ 数据库未连接 → 启动 PostgreSQL

---

### 问题 2: 提取任务中断

**症状**: 提取任务中途停止

**排查步骤**:

```bash
# Step 1: 检查进程
ps aux | grep extract_papers

# Step 2: 查看进度文件
cat tmp/extraction_progress_*.json | jq .

# Step 3: 查看日志末尾
```

### Q7: 如何备份和恢复数据？

**A**: 使用 PostgreSQL 工具

```bash
# 备份整个数据库
```

```bash

psql -U irriss_user irriss_leads < backup_20260315.sql

# 恢复特定表
psql -U irriss_user irriss_leads < paper_leads_backup.sql
```
